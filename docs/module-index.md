## Data access ‚Äî the truth about how the app reads data

- BigQuery client: [`src/lib/bq.ts`](../src/lib/bq.ts) ‚Üí `makeBQClient(projectId)`. Credentials from `GOOGLE_SERVICE_ACCOUNT_KEY` (inline JSON) or `GOOGLE_APPLICATION_CREDENTIALS` (keyfile). **The project id is passed per-call by the caller** ‚Äî it is not hardcoded in `bq.ts`.
- App data project: `muse-square-open-data` (EU). `ms-database-472505` is billing-only ‚Äî never queried for app data.
- Datasets observed in code: `semantic` (`vw_*` views ‚Äî the app's read layer), `mart` (`fct_*` facts), `raw` (source/ingested + user state), `analytics` (app-write state: logs, drafts, commitments, channel configs, sensitivity store), `dims` (`dim_*`), `open_data`.
- External services called: Anthropic API (`api.anthropic.com/v1/messages`), Browserless (`production-sfo.browserless.io`), Google Places / Business Profile, Meta Graph, Slack, Resend (email), BestTime, French BAN (`api-adresse.data.gouv.fr`), dbt Cloud (`ym384.us1.dbt.com`).
